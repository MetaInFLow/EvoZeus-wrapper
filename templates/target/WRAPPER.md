## Evolution Loop

1. User reports an unsatisfactory Skill result with the feedback Issue template.
2. Maintainer writes a design doc in `docs/designs/`.
3. Maintainer opens a PR referencing the design doc.
4. PR updates `SKILL.md` and `CHANGELOG.md`.
5. Release tag and release description are created after merge.
